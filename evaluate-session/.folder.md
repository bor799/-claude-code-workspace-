# evaluate-session

> **地位**: 会话质量审计员，在对话结束后量化 Claude 的表现，驱动持续改进
> **逻辑**: 读取会话历史 → 六维度打分（各 20/20/20/15/15/10%） → 减去红旗扣分 → 输出 0-100 总分 + 三级更新建议
> **约束**: 评估必须诚实客观，忽略明确指令扣 20 分/次，需要重做扣 25 分/次，不粉饰太平

## 架构

```
evaluate-session/
├── SKILL.md              # 技能定义
├── scripts/              # 评估脚本
└── references/           # 评估框架
```

## 六维度评估框架

| 维度 | 权重 | 说明 |
|------|------|------|
| **Spec Stability** | 20% | 需求稳定性 |
| **First-Attempt Accuracy** | 20% | 首次执行准确性 |
| **Feedback Sentiment** | 20% | 反馈情感倾向 |
| **Autonomy Level** | 15% | 自主工作能力 |
| **Completion Quality** | 15% | 完成质量 |
| **Learning & Improvement** | 10% | 学习与改进 |

## 红旗警示（扣分项）

| 问题 | 扣分 |
|------|------|
| "Wrong path" 纠正 | -10/次 |
| 重复相同错误 | -15/次 |
| 用户重复解释 | -10/次 |
| 忽略明确指令 | -20/次 |
| 需要重做大量工作 | -25/次 |

## 输入

| 类型 | 说明 |
|------|------|
| 会话历史 | 自动读取当前会话 |
| 用户反馈 | 手动补充 |

## 输出

| 类型 | 说明 |
|------|------|
| 性能得分 | 0-100 分 |
| 问题分析 | 红旗问题列表 |
| 改进建议 | 具体优化方向 |
| 三级更新 | 自动更新记忆/配置 |
