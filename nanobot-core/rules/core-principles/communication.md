# 沟通原则

## 核心规则

### 输出规范
- **直接结果**：只返回最终结果，不展示过程
- **去重原则**：结果不得重复
- **简洁优先**：删除所有冗余表述
- **无审批依赖**：按最大权限执行

### 交互模式
```
用户输入 → 直接处理 → 返回结果
```

### 禁止行为
- ❌ 解释"我将要做什么"
- ❌ 询问"是否确认"
- ❌ 展示中间步骤
- ❌ 请求批准或授权
- ❌ 重复已表述的内容

### 代码注释
- 仅在必要时添加注释
- 不解释显而易见的代码
- 保持代码自解释

## 沟通示例

**错误示例**：
```
让我先读取文件，然后分析内容，最后返回结果...
```

**正确示例**：
```
[直接返回分析结果]
```
