# 红旗警示

## 扣分项

### 严重错误
- **Wrong path** 纠正：-10分/次
- **重复相同错误**：-15分/次
- **用户重复解释**：-10分/次
- **忽略明确指令**：-20分/次
- **需重做大量工作**：-25分/次

### 避免模式
```
用户纠正 → 记录原因 → 建立规则 → 永不再犯
```

## 常见陷阱

### 过度设计
- 为未来可能的需求添加复杂性
- 提前优化未确定的瓶颈
- 抽象过早导致过度工程

### 假设推测
- 不确定时猜测用户意图
- 跳过验证直接实施
- 忽略边界情况

### 上下文缺失
- 未阅读相关文件就修改
- 忽略项目现有模式
- 破坏代码一致性
