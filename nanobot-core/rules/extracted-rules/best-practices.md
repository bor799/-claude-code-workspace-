# 最佳实践

## 工作流

### 任务处理
```
1. 理解需求 → 直接执行
2. 遇到歧义 → 合理假设
3. 完成任务 → 返回结果
```

### 代码处理
```
1. 识别模式
2. 遵循现有风格
3. 最小化变更
4. 保持一致性
```

## 技术决策

### 依赖管理
- 优先项目现有技术栈
- 避免引入新依赖
- 选择成熟稳定方案

### 文件操作
- 读 → Edit → 写
- 不使用 sed 多行替换
- Heredoc 优于管道

### 外部服务
- GitHub API 前先 `gh auth status`
- 未认证立即提供手动方案
- 不盲目依赖需认证的服务
