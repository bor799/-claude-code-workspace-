# subagents

> **地位**: 专业代理池，存放可独立调度的 Subagent 定义，每个代理有专属系统提示和工具权限
> **逻辑**: Claude 遇到复杂任务 → 委派给专业 Subagent → Subagent 在隔离上下文中执行 → 返回结果给主对话
> **约束**: 每个 Subagent 定义文件必须包含 description + 工具清单 + 执行指令，工具权限必须显式声明

## 架构

```
subagents/
├── agent-name-1.md       # Subagent 定义文件
├── agent-name-2.md       # Subagent 定义文件
└── ...
```

## Subagent 结构

每个 Subagent 定义文件包含：

```markdown
---
description: 代理描述
color: 颜色标识
---

# 代理名称

## 职责
...

## 工具
- tool1
- tool2

## 指令
...
```

## 输入

| 类型 | 说明 |
|------|------|
| 代理请求 | "调用 xxx 代理" |
| 任务委派 | Claude 主动委派复杂任务 |

## 输出

| 类型 | 说明 |
|------|------|
| 执行结果 | 代理完成任务的结果 |
| 状态报告 | 执行状态和中间结果 |

## 使用场景

- 复杂任务的专业处理
- 需要独立系统提示的任务
- 工具权限隔离的场景

## 核心文件

| 文件 | 职责 |
|------|------|
| `*.md` | Subagent 定义文件 |

## 注意事项

1. **独立上下文**: 每个 Subagent 有独立的系统提示
2. **工具隔离**: 可限制特定工具的访问
3. **任务委派**: Claude 可主动委派任务给 Subagent
