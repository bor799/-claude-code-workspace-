# doc-manager

> **地位**: 文档质量守护者，确保代码和文档永远保持同步，是 FractalFlow 协议的执行引擎之一
> **逻辑**: 接收代码变更信号 → 识别受影响的文档 → 按模板生成/更新 → 输出结构化 Markdown 文档
> **约束**: 代码变更后必须同步更新文档（原子更新规则），文档面向读者而非写作者，版本号与代码同步

## 架构

```
doc-manager/
├── SKILL.md              # 技能定义
├── scripts/              # 文档生成脚本
└── templates/            # 文档模板
```

## 文档类型

| 类型 | 说明 | 模板 |
|------|------|------|
| **技术文档** | 功能说明、API 参考 | tech-doc.md |
| **架构文档** | 系统架构、设计决策 | arch-doc.md |
| **README** | 项目说明、快速开始 | readme.md |
| **CLAUDE.md** | Claude Code 配置 | claude-md.md |

## 输入

| 类型 | 示例 |
|------|------|
| 代码库 | "为这个项目生成 README" |
| API 定义 | "生成 API 文档" |
| 架构图 | "记录系统架构" |

## 输出

| 类型 | 说明 |
|------|------|
| Markdown 文档 | 结构化的技术文档 |
| 图表 | 架构图、流程图 |
| 维护建议 | 文档更新建议 |

## 核心文件

| 文件 | 职责 |
|------|------|
| `SKILL.md` | 技能定义，文档规范 |

## 使用场景

- 新项目初始化文档
- API 变更更新文档
- 架构决策记录
- 项目知识库维护

## 注意事项

1. **及时更新**: 代码变更后同步更新文档
2. **清晰易懂**: 面向不同读者（开发者/用户）|
3. **版本控制**: 文档与代码版本同步

## ⚠️ 触发器

一旦本文件夹增删文件或架构逻辑调整，请立即重写此文档。
