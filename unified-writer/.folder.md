# unified-writer

> **地位**: 写作全流程的中央调度器，从内容获取到多平台发布的一条龙指挥中心
> **逻辑**: 接收写作/格式化/配图/发布请求 → 自动路由到最佳子工具 → 串联执行 → 输出成品或发布状态
> **约束**: AI 自主决策路由，禁止让用户手动指定工具；格式化必须先于发布；发布前必须确认内容

## 架构

```
统一写作入口 (unified-writer)
    │
    ├── 内容获取 → Jina Reader, agent-reach
    ├── 格式化 → markdown-to-html, format-markdown
    ├── 转换 → danger-x-to-markdown
    ├── 视觉生成 → image-gen, cover-image, infographic
    ├── 图片处理 → compress-image
    └── 发布 → post-to-wechat, post-to-x
```

## 输入

| 类型 | 示例 |
|------|------|
| 写作请求 | "写一篇关于 AI 的文章" |
| 格式化 | "格式化这篇 Markdown" |
| 配图 | "为文章生成配图" |
| 发布 | "发布到微信公众号" |

## 输出

| 类型 | 说明 |
|------|------|
| 文章内容 | Markdown/HTML 格式 |
| 视觉内容 | 图片、信息图、封面 |
| 发布状态 | 发布成功/失败反馈 |

## 子工具路由

| 任务 | 路由到 |
|------|--------|
| 读取网页 | agent-reach / Jina Reader |
| Markdown 格式化 | format-markdown |
| 转换为 HTML | markdown-to-html |
| 生成封面图 | cover-image |
| 压缩图片 | compress-image |
| 发布微信 | post-to-wechat |
| 发布 X | post-to-x |

## 核心文件

| 文件 | 职责 |
|------|------|
| `SKILL.md` | 技能定义，路由逻辑 |
| `scripts/` | 子工具脚本 |

## AI 自主决策

AI 根据用户请求自动选择子工具，无需手动指定：

```
用户: "写一篇文章并发布到微信"
    ↓
AI: 1. 调用写作工具生成内容
    2. 调用格式化工具处理
    3. 调用图片工具生成配图
    4. 调用发布工具推送到微信
```

## 注意事项

1. **智能路由**: AI 自动选择最佳工具
2. **全流程支持**: 从获取到发布一条龙
3. **多平台**: 微信、X、博客等
