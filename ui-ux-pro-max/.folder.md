# ui-ux-pro-max

> **地位**: 前端设计大脑，将自然语言需求转化为可执行的设计系统 + 前端代码，让非设计师也能产出专业界面
> **逻辑**: 接收设计需求/产品描述 → 搜索 67 风格/96 配色/57 字体 → 输出完整设计系统 + 可直接使用的代码
> **约束**: 严禁用 emoji 做 UI 图标（用 SVG）、严禁遗漏 cursor-pointer、暗色模式文字对比度必须 ≥ 4.5:1

## 架构

```
ui-ux-pro-max/
├── SKILL.md              # 技能定义
├── scripts/              # 核心脚本
├── data/                 # 设计数据
│   ├── styles/           # 67 种风格定义
│   ├── palettes/         # 96 种配色方案
│   ├── fonts/            # 57 种字体组合
│   └── charts/           # 25 种图表类型
└── design-system/        # 设计系统参考
```

## 设计能力

### 风格 (67 种)
- 编辑风、像素风、水彩、素描、蓝图、黑板
- 赛博朋克、复古、极简、温暖、自然等

### 配色 (96 种)
- 冷色、暖色、单色、粉彩、复古、鲜艳等

### 技术栈 (13 种)
- React, Next.js, Vue, Svelte, SwiftUI
- React Native, Flutter, Tailwind, shadcn/ui

## 输入

| 类型 | 示例 |
|------|------|
| 设计需求 | "设计一个登录页面" |
| 风格指定 | "使用赛博朋克风格" |
| 技术栈 | "用 Tailwind CSS" |

## 输出

| 类型 | 说明 |
|------|------|
| 设计方案 | 详细的设计规范 |
| 代码实现 | 可直接使用的前端代码 |
| 设计系统 | 完整的组件库 |

## 核心文件

| 文件 | 职责 |
|------|------|
| `SKILL.md` | 技能定义 |
| `data/styles/` | 风格定义 |
| `data/palettes/` | 配色方案 |
| `scripts/` | 核心脚本 |

## 使用流程

```
1. 用户描述设计需求
2. AI 分析并推荐风格/配色/字体
3. 生成设计方案 + 代码
4. 用户可迭代调整
```

## 注意事项

1. **非设计师友好**: 帮助发现审美偏好
2. **生产级代码**: 可直接使用
3. **避免 AI 美学**: 生成有特色的设计
