## 核心特质

### 工作风格
- **效率优先**: 不喜欢冗余解释，直接给结果
- **系统化思维**: 喜欢有框架、有评估体系的方法
- **主动学习**: 愿意尝试新工具，但要求有明确的价值
- **验证驱动**: 强调"验收标准"，要求系统可验证通过

### 编码偏好
- **命名**: snake_case for Python, camelCase for JS
- **注释**: 只在逻辑不显而易见时添加
- **重构**: 先理解现有模式，保持一致性
- **工具选择**: 严格遵循优先级（agent-reach → MCP → 内置）

### 决策模式
需要选择时：
- 提供选项 + 推荐 + 理由
- 遇到问题时：2-3个方案 + 推荐
- 不喜欢：直接说"看你决定"而不给建议

**任务类型识别关键词** (2026-03-06 更新):

- `优化` → 改进现有系统，分析后提出方案

### 工具偏好（更新）⚠️ 重要

### 新增观察
- **Skills 评估驱动设计**: 倾向于先评估 Skills 覆盖度（哪些已有、哪些需引入、哪些需自制），再制定计划

- **系统边界意识**: 需要理解不同系统的职责边界（如 NBO 负责调用，100X 负责萃取）
- **网络环境考虑**: API 调用需要考虑代理配置

- **清理策略**: 备份优先（删除前先移动到备份目录）
- **数据驱动决策**: 重视数据和评估结果，用分析来指导决策而非直觉
