# 用户画像 - Murphy

> 最后更新: 2026-03-27

## 核心特质

- **效率优先**: 不喜欢冗余解释，直接给结果
- **系统化思维**: 喜欢有框架、有评估体系的方法
- **验证驱动**: 强调"验收标准"，要求系统可验证通过

## 编码偏好

- **命名**: snake_case for Python, camelCase for JS
- **注释**: 只在逻辑不显而易见时添加
- **重构**: 先理解现有模式，保持一致性
- **工具选择**: 严格遵循优先级（见 rules/information-processing.md）

## 决策模式

需要选择时：
- 提供选项 + 推荐 + 理由
- 遇到问题时：2-3个方案 + 推荐
- 不喜欢：直接说"看你决定"而不给建议

## 反应触发点 ⚠️

**扣分项**:
- 重复相同错误 (-15分)
- 忽略明确指令 (-20分)
- 需要重做大量工作 (-25分)
- "Wrong path" 纠正 (-10分)
- 用户重复解释 (-10分)

## 沟通风格

- 简洁直接（手机交互）
- 查询无需批准，写入需批准
- 用表格/列表结构化信息
- 明确验收标准
- 不要过度工程
- 不确定时询问

## 品牌与设计偏好

- **顺丰数科**: 偏好清新配色（青绿色系 #0891B2, #22D3EE, #22C55E, #ECFEFF）
- **版权格式**: `© YYYY 部门名称 | 项目名称`
- **文案替换**: "销售" → "业务"
