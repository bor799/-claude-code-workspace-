# plan-feature

> **地位**: 功能开发的前期军师，在写代码之前规划路径，防止"想到哪写到哪"
> **逻辑**: 接收功能需求 → 拆解为任务清单 → 识别任务间依赖 → 评估风险 → 输出带优先级的实现策略
> **约束**: 每个任务必须可独立验证，风险评估必须包含缓解方案，禁止输出无法落地的空泛建议

## 架构

```
plan-feature/
├── SKILL.md              # 技能定义
├── scripts/              # 规划脚本
└── references/           # 规划模板
```

## 规划流程

```
需求分析 → 任务拆解 → 依赖识别 → 风险评估 → 实现策略
```

## 输入

| 类型 | 示例 |
|------|------|
| 功能描述 | "添加用户认证" |
| 需求文档 | PRD 链接或内容 |
| 技术栈 | 指定技术限制 |

## 输出

| 类型 | 说明 |
|------|------|
| 任务清单 | 按优先级排序 |
| 依赖关系 | 任务间的依赖图 |
| 风险列表 | 潜在风险 + 缓解方案 |
| 实现策略 | 推荐的技术路径 |

## 核心文件

| 文件 | 职责 |
|------|------|
| `SKILL.md` | 技能定义，规划框架 |

## 使用场景

- 新功能开发
- 复杂任务规划
- 技术方案设计
- PRD 解析

## 注意事项

1. **渐进式**: 支持分阶段实现
2. **风险意识**: 提前识别风险
3. **技术可行**: 评估技术可行性
