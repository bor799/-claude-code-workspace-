## 🛠️ 可用技能

### 技能 1: evaluate-session - 会话性能评估

**触发方式**:
- `/evaluate-session`
- "评估这次会话"
- "表现如何？"

**功能**:
- 基于五维度评估框架分析会话质量
- 识别红旗问题和改进机会
- 提供可执行的优化建议
- 识别可自动化的重复任务

**评估维度**:
1. Spec Stability (25%) - 需求稳定性
2. First-Attempt Accuracy (25%) - 首次执行准确性
3. Feedback Sentiment (20%) - 反馈情感倾向
4. Autonomy Level (15%) - 自主工作能力
5. Completion Quality (15%) - 完成质量

**适用场景**:
- ✅ 完成一个功能后总结
- ✅ 感觉效率低下想要改进
- ✅ 定期回顾工作流程
- ✅ 构建 CLAUDE.md 上下文

**示例使用**:
```
你: 评估这次会话的性能
AI: [会生成详细的性能评估报告，包括得分、问题分析、改进建议]
```

### 技能 2: code-review - 代码审查

**触发方式**:
- `/review [file_path]`
- "审查这段代码"
- "检查代码质量"

**功能**:
- 系统化代码审查（7个维度）
- 识别 bug、安全漏洞、性能问题
- 评估可维护性和可测试性
- 提供具体的改进建议和代码示例

**审查维度**:
1. ⚠️ 正确性 - 逻辑是否正确？
2. 🔒 安全性 - 是否有安全漏洞？
3. ⚡ 性能 - 是否高效？
4. 🔧 可维护性 - 代码是否清晰易懂？
5. 🧪 可测试性 - 是否易于测试？
6. 🎨 代码风格 - 是否一致？
7. 🏗️ 架构设计 - 是否合理？

**问题分类**:
- 🚨 阻断问题 - 必须修复
- ⚠️ 严重问题 - 应尽快修复
- 💡 改进建议 - 建议改进
- 📝 次要问题 - 可选优化
- ✨ 增强建议 - 未来考虑

**适用场景**:
- ✅ 审查 Pull Request
- ✅ 重构代码前评估
- ✅ 代码质量检查
- ✅ 学习最佳实践

**示例使用**:
```
你: 审查 src/user.service.ts
AI: [生成详细的代码审查报告，包括问题分类、修复建议、评分]
```

### 技能 3: plan-feature - 功能开发规划

**触发方式**:
- `/plan [功能描述]`
- "规划这个功能"
- "如何实现...？"

**功能**:
- 系统化需求分析
- 任务拆解（GRRAAP 框架）
- 依赖关系分析
- 风险评估和缓解策略
- MVP 和完整版规划

**规划框架**:
1. **需求分析** - 理解真正的问题
2. **任务拆解** - 按技术栈/阶段/数据流
3. **依赖分析** - 识别关键路径
4. **风险评估** - 技术/业务风险
5. **方案设计** - MVP 策略和技术选型

**输出内容**:
- 📋 需求概述
- 🎯 MVP 定义
- 📊 任务拆解表格
- 🔗 依赖关系图
- ⚠️ 风险评估矩阵
- 🛠️ 技术方案
- 📅 实施计划
- ✅ 验收标准

**适用场景**:
- ✅ 新功能开发
- ✅ 复杂任务规划
- ✅ 技术方案设计
- ✅ 项目重构规划
- ✅ 估算和排期

**示例使用**:
```
你: 规划一个文件上传功能
AI: [生成完整的功能规划报告，包括任务拆解、风险评估、实施计划]
```

## ❓ 常见问题

### Q: 如何查看所有可用技能？

使用命令查看技能列表：
```bash
ls -la ~/.claude/skills
```

### Q: 如何禁用某个技能？

删除符号链接：
```bash
rm ~/.claude/skills/skill-name
```

### Q: 如何创建自定义技能？

复制现有技能作为模板：
```bash
cp -r ~/.agents/skills/evaluate-session ~/.agents/skills/my-skill
cd ~/.claude/skills
ln -sf ../../.agents/skills/my-skill my-skill
```

### Q: 技能不生效怎么办？

1. 检查符号链接是否正确
2. 检查 skill.md 文件格式
3. 查看 Claude Code 日志

### Q: 如何调试技能？

在 skill.md 中添加调试输出，或使用简化版本测试
