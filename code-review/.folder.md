# code-review

> **地位**: 代码质量守门人，在开发完成后提供系统性审查，防止低质量代码进入代码库
> **逻辑**: 接收代码/PR → 四维度扫描（质量/安全/性能/可维护性） → 按严重度排序的问题清单 + 修复方案
> **约束**: 审查必须建设性，每条问题必须附带可操作的修复建议，禁止只批评不解决

## 成员清单

| 文件/目录 | 角色 | 说明 |

| `SKILL.md` | 入口 | 技能定义、审查流程、触发词 |
| `scripts/` | 执行器 | 审查辅助脚本 |
| `references/` | 知识库 | 审查标准、最佳实践 |

## 审查维度

| 维度 | 说明 | 检查项 |
|------|------|--------|
| **代码质量** | 命名、结构、可读性 | 函数长度、圈复杂度、命名规范 |
| **安全性** | 漏洞、注入、XSS | OWASP Top 10、敏感数据处理 |
| **性能** | 瓶颈、优化机会 | N+1 查询、内存泄漏、算法复杂度 |
| **可维护性** | 复杂度、耦合度 | 依赖关系、抽象层次、测试覆盖 |

## 输入/输出

| 类型 | 说明 |
|------|------|
| **输入** | 代码文件路径、PR 链接、代码片段 |
| **输出** | 问题清单（含严重度） + 修复方案 + 重构路径 |
