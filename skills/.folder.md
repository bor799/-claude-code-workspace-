# skills

> **地位**: 技能的孵化场，每个子目录是一个可被 Claude Code 调用的技能单元，与 shared/skills 双向共享
> **逻辑**: 用户触发词 → 匹配 SKILL.md → 加载技能定义 → 按流程执行 → 返回结果
> **约束**: 每个技能必须有 SKILL.md，触发词全局唯一，复杂技能分步披露而非一次全给

## 架构

```
skills/
├── skill-name-1/
│   └── SKILL.md          # 技能定义
├── skill-name-2/
│   └── SKILL.md          # 技能定义
└── ...
```

## 技能结构

每个技能目录包含：

```
skill-name/
├── SKILL.md              # 技能定义（必需）
├── scripts/              # 脚本文件（可选）
├── references/           # 参考文档（可选）
└── ...
```

## SKILL.md 格式

```markdown
---
description: 技能描述
---

# 技能名称

## 触发词
/trigger-word

## 职责
...

## 使用流程
...
```

## 输入

| 类型 | 示例 |
|------|------|
| 触发词 | "/skill-name" |
| 自然语言 | "帮我 xxx" |

## 输出

| 类型 | 说明 |
|------|------|
| 执行结果 | 技能执行的结果 |
| 交互提示 | 需要用户输入时的提示 |

## 核心文件

| 文件 | 职责 |
|------|------|
| `*/SKILL.md` | 技能定义 |

## 与 shared/skills 的关系

```
skills/          ← 工作区技能库
shared/skills/   ← 共享技能库
```

- 两个目录内容共享
- 修改任一目录，两边同时生效

## 注意事项

1. **SKILL.md 必需**: 每个技能必须有定义文件
2. **触发词唯一**: 避免冲突
3. **渐进式披露**: 复杂功能分步展示
