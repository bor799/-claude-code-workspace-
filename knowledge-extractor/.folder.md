# knowledge-extractor

> **地位**: 100X 知识萃取系统的 CLI 网关，Claude/Nanobot 通过它将链接推送到后台队列，真正的处理引擎在 Obsidian 目录
> **逻辑**: 接收 URL → 推送到 100X V2 队列（python -m src.main --add-url）→ AI 任务完成立即退出 → 后台守护进程自动处理
> **约束**: AI 只是生产者（推链接后立即退出），严禁等待处理完成、严禁终止守护进程、严禁自己跑 main.py

> ⚠️ **注意**: 这不是知识萃取系统本身，而是调用路由层。
> 真正的处理引擎位于: `~/Documents/Obsidian Vault/职业发展/项目案例/100X_知识萃取系统/knowledge-extractor/v2/`

## 数据流

```
Claude Code 用户发送链接
    ↓
AI 识别为需要萃取的内容
    ↓
执行: cd 100X/v2 && python -m src.main --add-url "URL"
    ↓
链接进入队列（AI 任务完成，退出）
    ↓
后台守护进程自动处理
    ↓
输出: Telegram 推送 + Obsidian 存储
```


## 重要注意事项

### AI 的角色（生产者模式）

**✅ 正确**:
- AI 只负责推送链接到队列
- 推送完成后立即退出
- 不等待处理完成
- 不干预守护进程

**❌ 错误**:
- 等待萃取结果
- 终止正在运行的进程
- 尝试自己完成整个处理流程

### 为什么需要这个网关？

1. **路径简化**: 100X 系统在 Obsidian 目录，路径很长
2. **集中管理**: 统一的调用入口
3. **环境隔离**: 100X 有独立的 Python 虚拟环境
4. **多入口支持**: Claude Code、Nanobot、手动调用
