# Knowledge Extractor - 智能知识萃取 v3.0

深度分析任何内容（长文、短消息、推特等），提取"洞见"和"高价值信息"，生成结构化情报报告。

---

## 执行方式

### URL 分析（推荐）

调用 100X 主流程进行深度分析：

```bash
cd "/Users/murphy/Documents/Obsidian Vault/职业发展/项目案例/100X_知识萃取系统/knowledge-extractor"
python src/main.py --url "<URL>" --verbose
```

**选项**：
- `--url`: 要分析的 URL（必需）
- `--verbose`: 显示详细输出（推荐）
- `--no-write`: 不写入 Obsidian（仅查看分析结果）
- `--dry-run`: 完整的干运行模式（不写入、不推送）

### 纯文本分析

对于无法通过 URL 访问的内容，使用下面的 LLM Prompt。

## Role 定义

**首席情报分析师 & 认知科学家**（10年经验）

擅长穿透表层信息，识别：
- 底层逻辑和思维模型
- 商业内幕和潜规则
- 反常识洞察
- 可复用方法论

---

## Goal

**深度知识萃取** — 提取"洞见"和"高价值信息"，而非平庸摘要。

关注：
- 水下信息（反常识、踩坑、内幕、数据）
- 高思想密度金句
- 可复用的方法论框架

---

## 核心输出结构

### 1. 重构标题
- **格式**：对象+问题+核心结论
- **限制**：< 20字
- **示例**："AB实验的统计陷阱与分层策略"

### 2. 一句话归纳
- **格式**：背景-冲突-解决方案
- **限制**：< 100字
- **示例**："背景：AB实验是产品优化的核心。冲突：多数实验存在统计谬误。方案：采用样本量计算和分层策略。"

### 3. 水下信息
- 反常识结论
- 踩坑经验
- 行业内幕
- 具体数据支撑

### 4. 金句萃取
- 高思想密度原句
- 可独立引用
- 压缩智慧

### 5. 方法论建模
- 可复用框架
- 工具栈推荐
- SOP步骤
