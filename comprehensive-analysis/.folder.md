# comprehensive-analysis

> **地位**: 投资决策的大脑，整合 5 个专业 Agent 分析任何资产，回答"该不该买、什么时候买"的核心问题
> **逻辑**: 接收公司名/代码/加密货币 → 并行调用 2-5 个 Agent → 综合研判 → 输出买入/观望/卖出建议 + 信心水平
> **约束**: 必须回答 3 个核心问题（最关键价值点？现在该买吗？买入信号？），不限于科技股，不提供保证收益

## 架构

```
comprehensive-analysis/
├── SKILL.md              # 技能定义文档
├── agents/               # 专业 Agent 集合
│   ├── tech-earnings.md      # 财报深度分析
│   ├── value-investing.md    # 价值投资评估
│   ├── macro-liquidity.md    # 宏观流动性
│   ├── market-sentiment.md   # 市场情绪
│   └── crypto-bottom.md      # 加密货币底部
└── references/           # 参考资料
```

## 5 个专业 Agent

| Agent | 职责 | 适用场景 |
|-------|------|----------|
| **tech-earnings** | 财报深度分析 | 季报、年报分析 |
| **value-investing** | 价值投资评估 | 长期持有决策 |
| **macro-liquidity** | 宏观流动性 | 市场环境判断 |
| **market-sentiment** | 市场情绪 | 短期波动分析 |
| **crypto-bottom** | 加密货币底部 | 加密市场抄底 |

## 输入

| 类型 | 示例 |
|------|------|
| 公司名称 | "分析 Tesla" |
| 股票代码 | "分析 AAPL" |
| 加密货币 | "分析 BTC 是否到底部" |
| 具体问题 | "NVDA 值得买入吗？" |

## 输出

| 类型 | 说明 |
|------|------|
| 综合报告 | 整合 5 个 Agent 的分析结果 |
| 买入建议 | 明确的买入/观望/卖出建议 |
| 风险提示 | 潜在风险因素 |

## 使用流程

```
1. 用户触发 /comprehensive-analysis "公司名"
2. AI 自动选择相关 Agent
3. 并行执行分析
4. 整合结果
5. 输出综合报告 + 买入建议
```

## 外部依赖

- **agent-reach**: 互联网搜索（财报、新闻）
- **知识萃取**: 分析相关文章
- **Claude Opus 4.6**: 分析引擎

## 注意事项

1. **不限于科技股**: 支持任何上市公司分析
2. **交互式**: 可针对分析结果追问
3. **多维度**: 整合基本面、技术面、宏观、情绪
